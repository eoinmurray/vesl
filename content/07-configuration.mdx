# Configuration

veslx works without config. Values are derived from `package.json` and git.

## Zero Config Defaults

- `name` → from `package.json` or folder name
- `shortName` → first 2 chars of name
- `description` → from `package.json`
- `github` → parsed from git remote

## Optional Config

Create `veslx.yaml` to override:

```yaml
dir: docs
site:
  name: My Project
  github: user/repo
```

## Generate

```bash
veslx init
```

## Options

### `dir`

Content directory.

```yaml
dir: docs
```

Default: `.`

### `site.name`

Site name for header and title.

```yaml
site:
  name: My Docs
```

Default: `package.json` name or folder name

### `site.shortName`

Header logo text.

```yaml
site:
  shortName: MD
```

Default: first 2 chars of name

### `site.description`

Homepage subtitle.

```yaml
site:
  description: Documentation
```

Default: `package.json` description

### `site.github`

GitHub link in header.

```yaml
site:
  github: user/repo
```

Default: parsed from `git remote get-url origin`
